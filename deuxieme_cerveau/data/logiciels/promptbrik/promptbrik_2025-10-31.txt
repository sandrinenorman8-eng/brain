19:00:55: Délimiteurs Hiérarchiques (Évite la Fatigue) :

Parenthèses rondes () : Pour sous-notes ou clarifications mineures (ex. : (niveau: EXPERTLEVEL)). Limite à 1-2 niveaux ; au-delà, passe à crochets.
Crochets [] : Pour sections majeures ou listes actions (ex. : [optimisation,enrich,repare]). Idéal pour objectifs, car visuellement distinct.
Accolades {} : Pour formats de sortie obligatoires (ex. : {output: JSON avec clés}). Utilisé en 2025 pour agentic workflows (Medium guide).
Guillemets triples """ ou XML tags < > : Pour blocs texte/input (ex. : """Texte à analyser"""). Recommandé par LangChain pour séparer user input de system policy – évite confusion dans multi-turns.
Deux-points : : Pour transitions logiques (ex. : Task: Analyse). Pas pour imbrication ; garde-le linéaire.
Best Practices pour Accentuation (Bacancy Technology, 2025) :

Sépare tout : System Policy || Task || Input. Ex. : Utilise |||, ---, ou lignes vides pour diviser.
Chain of Thought (CoT) Imbriqué : Pour complexité, guide pas à pas : "Étape 1: [Identifier problème]. Étape 2: (Proposer solution)."
One-Shot Examples : Ajoute 1 exemple clair pour "démontrer" ta notation. Ex. : "Accentue comme ceci: [IMPORTANT: Action clé]."
Évite Pitfalls : Pas de sur-imbrication (ton original en a 3+ niveaux → LLM peut ignorer). Pas de majuscules excessives (fatigue parsing). Teste itérativement (LinkedIn post 2025 : "Instruis le modèle à vérifier assumptions d'abord").
Pour 2025 Models (OpenAI updates) : Les LLMs comme o3/o4-mini intègrent mieux les "system instructions" séparées, donc mets ton rôle en haut comme [SYSTEM: ...].
